# KF-002: Add CLI command to set fleet orchestration turn limit

## PURPOSE

Operators currently can't tune orchestration turn limits without editing config or code. Expose a CLI command so the limit can be changed quickly per-environment without redeploying.

## CURRENT_STATE

null

## REQUIREMENTS

- New CLI subcommand (e.g. `kyberbot fleet set-turns <N>`) that updates the persisted limit.
- Companion read command (e.g. `kyberbot fleet get-turns`) to inspect the current value.
- Validation: N must be a positive integer; reject non-numeric or zero/negative input with a clear error.
- Value persists across restarts (store wherever fleet orchestration config currently lives).
- New value takes effect on the next orchestration run without restart, if feasible — otherwise document the restart requirement.
- Update CLI `--help` output and any relevant docs.
